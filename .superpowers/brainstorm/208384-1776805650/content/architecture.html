Proposed Architecture

Habit tracker follows the same Clean Architecture pattern as todo feature

Package Structure: habit/
habit/ ├── data/ │ ├── repository/ │ │ └── HabitRepositoryImpl.kt # Android: Room, iOS: UserDefaults │ └── mapper/ │ └── HabitMapper.kt ├── domain/ │ ├── model/ │ │ └── Habit.kt # id, name, recurrenceType, daysOfWeek, color │ ├── repository/ │ │ └── HabitRepository.kt # interface │ └── usecase/ │ ├── GetHabitsForDayUseCase.kt │ ├── AddHabitUseCase.kt │ ├── ToggleHabitCompletionUseCase.kt │ └── GetHabitHistoryUseCase.kt ├── presentation/ │ ├── screen/ │ │ └── HabitScreen.kt │ ├── viewmodel/ │ │ └── HabitViewModel.kt │ └── composable/ │ ├── HabitItem.kt │ └── AddHabitDialog.kt └── di/ └── HabitModule.kt

Consistent with todo

  • Same layer pattern: domain → data → presentation
  • Same DI module structure
  • Same naming conventions
  • Easy to maintain and understand

Platform split

  • Room for Android persistence
  • UserDefaults for iOS persistence
  • Both behind shared repository interface

Does this architecture look right for the habit feature?
